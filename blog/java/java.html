1.一个".java"源文件中是否可以包括多个类（不是内部类）？有什么限制？

在Java中，一个源文件（.java文件）可以包含多个类，但只能有一个公共类（public class）。公共类的名称必须与文件名相同，以便编译器能够正确识别。

其他非公共类可以在同一个文件中定义，但不能声明为public。这些非公共类可以被同一文件中的其他类使用，但在其他文件中无法直接访问。

需要注意的是，虽然可以在同一文件中定义多个类，但最佳实践是将每个类放入单独的文件中，使代码结构更清晰和易于维护。

2.Java有没有goto?

在标准的Java语言中，并没有提供类似于C语言中的goto语句。Java设计者在语言设计时故意避免了goto语句的引入，主要出于以下几个原因：

  1. 简化控制流：使用goto语句可能导致程序的控制流变得复杂和难以理解。通过使用结构化的控制流语句（如if、for、while等），可以更清晰地表达程序逻辑，降低代码维护的复杂性。
  2. 安全性考虑：goto语句可能导致跳转到程序中任意位置，容易产生不可预测的行为，使程序变得难以调试和维护。Java强调安全性和可靠性，限制了这种不受限制的跳转。
  3. 结构化编程：Java鼓励使用结构化编程范式，将代码分解为模块化的、可重用的部分，并通过方法调用、异常处理等机制来组织程序逻辑。这种方式更有利于代码的理解和维护。

尽管Java本身不支持goto语句，但可以通过其他控制流语句（如break、continue、return等）和代码重构来实现相同的功能，同时保持代码的可读性和可维护性。

3.说说&和&&的区别

&和&&都可以用作逻辑与的运算符，表示逻辑与（and），当运算符两边的表达式的结果都为true时，整个运算结果才为true，否则，只要有一方为false，则结果为false。

&&还具有短路的功能，即如果第一个表达式为false，则不再计算第二个表达式，例如，对于if(str != null && !str.equals(“”))表达式，当str为null时，后面的表达式不会执行，所以不会出现NullPointerException如果将&&改为&，则会抛出NullPointerException异常。If(x==33 & ++y>0) y会增长，If(x==33 && ++y>0)不会增长。

&还可以用作位运算符，当&操作符两边的表达式不是boolean类型时，&表示按位与操作，我们通常使用0x0f来与一个整数进行&运算，来获取该整数的最低4个bit位，例如，0x31 & 0x0f的结果为0x01。

5.switch语句能否作用在byte上，能否作用在long上，能否作用在String上?

 

在Java中，switch语句可以作用在byte、short、int和char等整数类型上，由于byte,short,char都可以隐含转换为int。但不能直接作用在long类型上，这是因为long类型占用的字节数较大，不符合switch语句的要求。

然而，从Java 7开始，可以在switch语句中使用String类型。这使得我们可以根据不同的字符串值执行相应的操作，提供了更灵活的编程选项。

7.char型变量中能不能存贮一个中文汉字?为什么?

在Java中，char类型是用于表示Unicode字符的数据类型，每个char变量占用16位（2个字节）的存储空间。由于Unicode字符集可以表示包括中文汉字在内的各种字符，因此理论上char类型是可以存储一个中文汉字的。

然而，需要注意的是，Java中的char类型使用的是UTF-16编码，而大部分中文汉字使用的是Unicode编码，其中一些汉字的编码超过了char类型的表示范围。UTF-16编码使用了变长编码，对于超出char类型表示范围的字符，需要使用两个char单元来表示。

因此，虽然char类型可以存储一些中文汉字，但不能完全涵盖所有的中文字符。如果要处理包含全面的中文字符集的文本，应该使用Java中的字符串类型（String），因为字符串类型可以存储任意Unicode字符序列，包括中文汉字。

8.用最有效率的方法算出2乘以8等于几?

对于乘法运算，最有效率的方法是使用位运算来实现。对于乘法运算2乘以8，可以使用左移运算符（<<）来完成。

xxxxxxxxxx
int result = 2 << 3;

在这里，将2的二进制表示左移3位，等效于将2乘以2的3次方（即2的立方），得到的结果为16。因此，2乘以8的结果是16。

使用位运算的方式进行乘法运算可以减少计算的时间和资源消耗，因为位运算是基于底层位操作的快速操作。然而，在实际编码中，也应该根据代码的可读性和可维护性来权衡使用位运算的效益。

10.使用final关键字修饰一个变量时，是引用不能变，还是引用的对象不能变？

当使用final关键字修饰一个变量时，是指该变量的引用不能变，即变量只能被赋值一次，赋值后不能再指向其他对象。这意味着你不能将final修饰的变量再次赋值为其他对象。

然而，final并不表示被引用的对象本身不能发生变化。如果被引用的对象是可变的，那么通过final修饰的变量仍然可以修改该对象的内部状态。

以下是一个示例来说明这个概念：

xxxxxxxxxx
final StringBuilder sb = new StringBuilder("Hello");
sb.append(" World"); // 可以修改被引用对象的内部状态

在上述示例中，虽然sb变量使用了final修饰，但我们仍然可以调用append方法修改StringBuilder对象的内容。这是因为final修饰的是变量本身，而不是被引用的对象。

总结起来，final修饰的变量意味着引用不能变，而被引用对象的内部状态可以变化（对于可变对象）。对于不可变对象（如String类），使用final修饰的变量既表示引用不能变，也表示被引用对象本身是不可变的。

11."=="和equals方法究竟有什么区别？

"=="和equals方法是Java中用于比较对象的操作符和方法，它们有以下几个区别：

  1. 比较的对象类型：

    • "=="操作符：主要用于比较基本数据类型的值或比较引用类型的地址，即比较两个对象是否指向同一内存地址。
    • equals方法：主要用于比较引用类型的对象内容，即比较两个对象的内容是否相等。

  2. 实现方式：

    • "=="操作符：执行引用比较，检查两个操作数是否引用同一个对象。
    • equals方法：默认情况下，执行引用比较，与"=="操作符相同。但是，equals方法可以被类重写，以自定义对象相等的比较方式。

  3. 默认行为：

    • "=="操作符：对于基本数据类型，比较它们的值；对于引用类型，比较它们的引用地址。
    • equals方法：默认情况下，与"=="操作符的行为相同。对于Object类，equals方法的默认实现与"=="操作符的行为相同。

  4. 自定义比较：

    • "=="操作符：无法自定义比较方式。
    • equals方法：可以根据需要重写该方法，自定义对象之间相等的比较规则。在重写时，需要遵循equals方法的一致性、对称性、传递性和非空性等规范。

综上所述，"=="操作符用于比较基本类型的值和引用类型的地址，而equals方法用于比较引用类型对象的内容。对于自定义类，应该根据需求重写equals方法，以实现自定义对象相等的比较方式。

12.静态变量和实例变量的区别？

静态变量（也称为类变量）和实例变量是Java中两种不同类型的变量，它们具有以下区别：

  1. 存储位置：

    • 静态变量：静态变量属于类，它们在类加载时被分配内存空间，并且只有一个副本。静态变量存储在方法区中，并且在整个程序运行期间都存在。
    • 实例变量：实例变量属于类的实例（对象），每个对象都有一份实例变量的副本。实例变量存储在堆内存中，它们的生命周期与对象的生命周期相关。

  2. 访问方式：

    • 静态变量：可以直接通过类名访问，例如ClassName.staticVariable。静态变量可以在类的任何方法中访问，也可以在类外部通过类名访问。
    • 实例变量：必须通过对象引用访问，例如objectName.instanceVariable。实例变量只能在创建对象后才能访问。

  3. 生命周期：

    • 静态变量：静态变量在类加载时被初始化，一直存在于内存中，直到程序结束或类被卸载。
    • 实例变量：实例变量的生命周期与对象的生命周期相同。当创建对象时，实例变量被初始化；当对象不再被引用时，实例变量的内存会被垃圾回收器回收。

  4. 共享性：

    • 静态变量：静态变量被所有该类的对象共享，即使没有创建对象，也可以通过类名直接访问。
    • 实例变量：每个对象都有自己的一份实例变量的副本，它们相互独立，不共享。

  5. 用途：

    • 静态变量：通常用于表示与类相关的常量或共享的数据。例如，表示常量的PI值或记录创建对象的计数器。
    • 实例变量：用于表示对象的特定状态或属性，每个对象都有自己的一份实例变量。

需要根据具体需求和设计来选择使用静态变量还是实例变量。静态变量适用于表示与类相关的数据，实例变量适用于表示每个对象的独立状态或属性。

13.是否可以从一个static方法内部发出对非static方法的调用？

不可以。

因为非static方法是要与对象关联在一起的，必须创建一个对象后，才可以在该对象上进行方法调用，而static方法调用时不需要创建对象，可以直接调用。

也就是说，当一个static方法被调用时，可能还没有创建任何实例对象，如果从一个static方法中发出对非static方法的调用，那个非static方法是关联到哪个对象上的呢？这个逻辑无法成立，所以，一个static方法内部发出对非static方法的调用。

如果在静态方法内部需要访问非静态方法，可以通过创建对象实例来调用非静态方法。或者，将非静态方法设置为静态方法，以便可以直接从静态方法内部调用。

xxxxxxxxxx
public class MyClass {
    private int value;


    public static void main(String[] args) {
        MyClass myObject = new MyClass();
        myObject.nonStaticMethod(); // 通过对象实例调用非静态方法
    }


    public void nonStaticMethod() {
        // 非静态方法的实现
        System.out.println("Non-static method");
    }
}

在上述示例中，通过创建MyClass类的对象实例myObject，然后使用myObject.nonStaticMethod()来调用非静态方法nonStaticMethod()。这样就可以在静态方法main()中调用非静态方法。

14.Integer与int的区别

Integer和int是Java中表示整数的两种类型，它们之间有以下区别：

  1. 类型：

    • int：是Java的基本数据类型，直接存储整数值。
    • Integer：是int的包装类（Wrapper Class），它提供了将基本类型int封装为对象的功能。

  2. 装箱和拆箱：

    • int是基本类型，直接存储整数值，不需要装箱和拆箱的过程。
    • Integer是对象，它需要进行装箱（将int转换为Integer对象）和拆箱（将Integer对象转换为int）的操作。

  3. 空值处理：

    • int不能表示空值，它是一个原始数据类型，必须具有非零的初始值。
    • Integer可以表示空值。通过使用null赋值给Integer对象，可以表示一个没有值的情况。

  4. 功能和方法：

    • int是一个原始类型，因此不具有任何额外的功能或方法。
    • Integer作为一个类，提供了一些方法来操作整数，如比较、转换为字符串等。

需要注意的是，在使用Integer类型时，由于装箱和拆箱的操作会引入一定的性能开销，因此在性能敏感的情况下，应该优先考虑使用int类型。而在需要使用对象的上下文中，或需要使用空值表示的情况下，可以使用Integer类型。此外，Java提供了自动装箱和拆箱的特性，使得在使用过程中可以自动在int和Integer之间进行转换。

在JSP开发中，Integer的默认为null，所以用el表达式在文本框中显示时，值为空白字符串，而int默认的默认值为0，所以用el表达式在文本框中显示时，结果为0，所以，int不适合作为web层的表单数据的类型。

在Hibernate中，如果将OID定义为Integer类型，那么Hibernate就可以根据其值是否为null而判断一个对象是否是临时的，如果将OID定义为了int类型，还需要在hbm映射文件中设置其unsaved-value属性为0。

另外，Integer提供了多个与整数相关的操作方法，例如，将一个字符串转换成整数，Integer中还定义了表示整数的最大值和最小值的常量。

15.Math.round(11.5)等于多少? Math.round(-11.5)等于多少?

在Math类中提供了三个与取整有关的方法：ceil、floor、round。这些方法的作用与它们的英文名称的含义相对应。

  1. ceil（天花板）：该方法表示向上取整，即取最接近且大于等于原始值的整数。例如，Math.ceil(11.3)的结果是12，Math.ceil(-11.3)的结果是-11。
  2. floor（地板）：该方法表示向下取整，即取最接近且小于等于原始值的整数。例如，Math.floor(11.6)的结果是11，Math.floor(-11.6)的结果是-12。
  3. round（四舍五入）：该方法表示四舍五入取整，将原始值加上0.5后再向下取整。例如，Math.round(11.5)的结果是12，Math.round(-11.5)的结果是-11。

需要注意的是，round方法的算法是将原始值加上0.5后再向下取整。这是为了实现四舍五入的效果。

这些方法提供了不同的取整方式，可以根据需要选择合适的方法来进行取整操作。

在Java中，有四种访问修饰符（access modifiers）用于控制类、变量、方法和构造函数的可见性和访问权限：public、private、protected和默认（不写修饰符）。

  1. public：

    • 可以在任何位置访问具有public访问修饰符的成员（类、变量、方法、构造函数）。
    • public成员对于所有类和包都是可见的，可以被任意外部类访问。

  2. private：

    • private访问修饰符仅在所在类的内部可见，不能在类的外部访问。
    • private成员对于同一个类中的其他成员是可见的，但对于其他类是不可见的。

  3. protected：

    • protected访问修饰符对于同一个包中的其他类和所有子类是可见的。
    • protected成员可以在同一包中的其他类中进行访问，以及在继承关系中的子类中进行访问。

  4. 默认（不写修饰符）：

    • 默认访问修饰符（也称为包级访问或friendly）仅在同一个包中可见。
    • 默认成员对于同一个包中的其他类是可见的，但对于其他包中的类是不可见的。子孙类也不可见。

需要注意的是，这些访问修饰符可以用于类的成员，但不能用于顶层类（即只有public和默认可用于顶层类）。

正确选择适当的访问修饰符对于控制类的封装性、模块化和代码安全性是非常重要的。根据具体的设计和需求，选择合适的修饰符来限制成员的访问范围。

17.Overload和Override的区别。Overloaded的方法是否可以改变返回值的类型?

Overload（重载）和Override（重写）是Java中两个不同的概念，它们有以下区别：

  1. 定义：

    • Overload（重载）：指在同一个类中创建多个具有相同名称但参数列表不同的方法，以便根据不同的参数类型和个数进行方法的区分。
    • Override（重写）：指在子类中重新实现父类中已经定义的方法，方法名、参数列表和返回类型必须与父类中的方法相同。

  2. 发生的位置：

    • Overload（重载）：发生在同一个类中，即一个类内部。
    • Override（重写）：发生在子类和父类之间，即一个类继承另一个类。

  3. 基础方法：

    • Overload（重载）：可以在同一个类中定义多个具有相同名称但参数列表不同的方法，例如同名方法的重载。（当然，同一方法内的几个参数类型必须不一样，例如可以是fun(int,float)，但是不能为fun(int,int)）
    • Override（重写）：子类可以重写父类中已经定义的方法，以实现特定的行为。

  4. 特性：

    • Overload（重载）：重载方法根据参数的类型、个数或顺序进行区分，并可以具有不同的返回类型。返回类型不是用于重载方法的标识。Overloaded的方法的参数列表不一样，它们的返回者类型当然也可以不一样。但我估计你想问的问题是：如果两个方法的参数列表完全一样，是否可以让它们的返回值不同来实现重载Overload。这是不行的，我们可以用反证法来说明这个问题，因为我们有时候调用一个方法时也可以不定义返回结果变量，即不要关心其返回结果，例如，我们调用map.remove(key)方法时，虽然remove方法有返回值，但是我们通常都不会定义接收返回结果的变量，这时候假设该类中有两个名称和参数列表完全相同的方法，仅仅是返回类型不同（如int（表示删除的值）和boolean类型（表示删除是否成功），java就无法确定编程者倒底是想调用哪个方法了，因为它无法通过返回结果类型来判断。不能通过访问权限、返回类型、抛出的异常进行重载；

      对于继承来说，如果某一方法在父类中是访问权限是priavte，那么就不能在子类对其进行重载（因为私有方法在子类中无法访问到），如果定义的话，也只是定义了一个新方法，而不会达到重载的效果。

    • Override（重写）：重写方法必须具有相同的方法名、参数列表和返回类型。覆盖的方法所抛出的异常必须和被覆盖方法的所抛出的异常一致，或者是其子类；被覆盖的方法不能为private，否则在其子类中只是新定义了一个方法，并没有对其进行覆盖。

需要注意的是，Overloaded（重载）的方法可以具有不同的返回类型，因为方法重载是根据参数来区分的，而返回类型不是重载方法的标识。但是，在Override（重写）中，重写的方法必须具有与父类中被重写的方法相同的返回类型。

总结起来，Overload（重载）发生在同一个类中，可以根据参数的类型、个数或顺序进行方法的区分；Override（重写）发生在子类和父类之间，子类重新实现了父类中已经定义的方法，重写方法必须具有相同的方法名、参数列表和返回类型。Overloaded（重载）的方法可以具有不同的返回类型，但Override（重写）的方法必须具有相同的返回类型。

18.构造器Constructor是否可被override?

构造器（Constructor）不能被重写（Override）。

在Java中，构造器是用于创建对象的特殊方法，它与类同名且没有返回类型。构造器在创建对象时被调用，并用于初始化对象的实例变量和执行其他必要的操作。

重写（Override）是指在子类中重新实现父类中已经定义的方法，以实现特定的行为。重写要求子类的方法具有与父类方法相同的方法名、参数列表和返回类型。然而，构造器是与类同名的特殊方法，它没有返回类型，因此无法满足重写的条件。

虽然不能重写构造器，但是子类可以通过使用super()调用父类的构造器，以便在创建子类对象时执行父类的初始化操作。这被称为构造器的链式调用（constructor chaining）。

需要注意的是，如果子类没有明确定义构造器，则会默认继承父类的构造器，但这并不属于重写的范畴。子类的构造器和父类的构造器是相互独立的，它们在各自的类层级中起作用，没有重写的概念。

但可以被重载Overload（通过重载生成不同的对象）。

19.接口是否可继承接口? 抽象类是否可实现(implements)接口? 抽象类是否可继承具体类(concrete class)? 抽象类中是否可以有静态的main方法？

在Java中，接口、抽象类和具体类之间有一些规则和限制，它们的关系如下：

  1. 接口是否可继承接口：

    • 是的，接口可以继承其他接口，通过使用extends关键字来实现接口之间的继承关系。
    • 通过继承，子接口可以继承父接口的方法声明，从而扩展和定义新的方法。

  2. 抽象类是否可实现（implements）接口：

    • 是的，抽象类可以实现（implements）接口，通过使用implements关键字来表示抽象类对接口的实现。
    • 抽象类实现接口时，必须实现接口中的所有抽象方法，除非该抽象类本身也是一个抽象类。

  3. 抽象类是否可继承具体类（具体类即非抽象类）：

    • 是的，抽象类可以继承具体类（非抽象类），通过使用extends关键字来表示抽象类对具体类的继承。
    • 抽象类继承具体类时，可以继承具体类的字段、方法和非私有的内部类。

  4. 抽象类中是否可以有静态的main方法：

    • 是的，抽象类中可以有静态的main方法。
    • 静态的main方法是程序的入口点，用于启动Java应用程序。在抽象类中定义main方法可以作为程序的入口，并在需要时调用其他方法或创建对象。

需要注意的是，一个类可以同时实现多个接口，但只能继承一个类（包括抽象类和具体类）。通过接口的多继承和抽象类的继承，可以创建具有更灵活结构的类层次结构。

抽象类（Abstract Class）是在面向对象编程中用来表示具有抽象（未完全实现）方法的类。它是一种特殊的类，无法被直接实例化，而是作为其他具体子类的基类或模板。

抽象类通过使用abstract关键字进行声明。它可以包含抽象方法（没有实现）和具体方法（有实现），以及字段、构造函数和其他普通方法。抽象方法用于定义类的接口，而具体方法提供了一些默认实现。

抽象类的主要特点包括：

  1. 不能被直接实例化：由于抽象类包含未实现的抽象方法，因此无法创建抽象类的实例。需要通过创建其具体子类的实例来使用抽象类。
  2. 可以被继承：其他类可以通过继承抽象类来扩展其功能。子类必须实现抽象类中的所有抽象方法，或者也可以将自己定义为抽象类。
  3. 可以包含抽象方法：抽象方法没有具体的实现，只有方法的声明。子类必须实现抽象类中的所有抽象方法，或者也可以将自己定义为抽象类。

抽象类常用于以下情况：

  • 定义一组类的共性行为和方法，并将这些方法留给具体子类实现。
  • 提供一些默认实现，减少子类的代码重复。
  • 作为设计框架的基础，通过继承和实现来扩展和定制具体的功能。

只有记住抽象类与普通类的唯一区别就是不能创建实例对象和允许有abstract方法。

21.面向对象的特征有哪些方面

计算机软件系统是现实生活中的业务在计算机中的映射，而现实生活中的业务其实就是一个个对象协作的过程。

面向对象编程就是按现实业务一样的方式将程序代码按一个个对象进行组织和编写，让计算机系统能够识别和理解用对象方式组织和编写的程序代码，这样就可以把现实生活中的业务对象映射到计算机系统中。

面向对象的编程语言有封装、继承 、抽象、多态等4个主要的特征。

1）封装：

封装是保证软件部件具有优良的模块性的基础，封装的目标就是要实现软件部件的“高内聚、低耦合”，防止程序相互依赖性而带来的变动影响。在面向对象的编程语言中，对象是封装的最基本单位，面向对象的封装比传统语言的封装更为清晰、更为有力。面向对象的封装就是把描述一个对象的属性和行为的代码封装在一个“模块”中，也就是一个类中，属性用变量定义，行为用方法进行定义，方法可以直接访问同一个对象中的属性。

通常情况下，只要记住让变量和访问这个变量的方法放在一起，将一个类中的成员变量全部定义成私有的，只有这个类自己的方法才可以访问到这些成员变量，这就基本上实现对象的封装，就很容易找出要分配到这个类上的方法了，就基本上算是会面向对象的编程了。把握一个原则：把对同一事物进行操作的方法和相关的方法放在同一个类中，把方法和它操作的数据放在同一个类中。

2）继承：

在定义和实现一个类的时候，可以在一个已经存在的类的基础之上来进行，把这个已经存在的类所定义的内容作为自己的内容，并可以加入若干新的内容，或修改原来的方法使之更适合特殊的需要，这就是继承。继承是子类自动共享父类数据和方法的机制，这是类之间的一种关系，提高了软件的可重用性和可扩展性。

3）抽象：

抽象就是找出一些事物的相似和共性之处，然后将这些事物归为一个类，这个类只考虑这些事物的相似和共性之处，并且会忽略与当前主题和目标无关的那些方面，将注意力集中在与当前目标有关的方面。例如，看到一只蚂蚁和大象，你能够想象出它们的相同之处，那就是抽象。抽象包括行为抽象和状态抽象两个方面。例如，定义一个Person类，如下：

class Person{

String name;

int age;

}

人本来是很复杂的事物，有很多方面，但因为当前系统只需要了解人的姓名和年龄，所以上面定义的类中只包含姓名和年龄这两个属性，这就是一种抽像，使用抽象可以避免考虑一些与目标无关的细节。我对抽象的理解就是不要用显微镜去看一个事物的所有方面，这样涉及的内容就太多了，而是要善于划分问题的边界，当前系统需要什么，就只考虑什么。

4）多态：

多态是指程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定，而是在程序运行期间才确定，即一个引用变量到底会指向哪个类的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须在由程序运行期间才能决定。

因为在程序运行时才确定具体的类，这样，不用修改源程序代码，就可以让引用变量绑定到各种不同的类实现上，从而导致该引用调用的具体方法随之改变，即不修改程序代码就可以改变程序运行时所绑定的具体代码，让程序可以选择多个运行状态，这就是多态性。

多态是面向对象编程中的一个重要概念，指的是同一类型的对象在不同情境下可以表现出不同的行为或状态。简而言之，多态性允许我们使用统一的接口来处理不同类型的对象。

在面向对象编程语言中，多态性通常通过继承和方法重写来实现。当一个父类有多个子类时，可以使用父类的引用变量来引用子类的对象，而通过这个引用变量调用的方法实际上是子类中重写的方法。

具体来说，多态有两个关键要素：

  1. 继承关系：多态性需要存在父类和子类之间的继承关系。子类继承了父类的属性和方法，同时可以重写父类的方法或添加新的方法。
  2. 方法重写：子类可以对父类中的方法进行重写，即在子类中重新定义和实现父类中已经存在的方法。通过方法重写，子类可以根据自身的特性来改变方法的行为。

多态性的好处在于可以提高代码的灵活性和可扩展性。通过使用多态性，可以编写通用的代码，可以处理多种不同类型的对象，而无需针对每种对象都编写特定的代码。这样可以减少代码的重复性，提高代码的可维护性和可复用性。

举个例子来说明多态性的概念：

假设有一个动物类（Animal），它有一个方法叫作"发出声音"（makeSound）。现在有两个子类，狗类（Dog）和猫类（Cat），它们都继承自动物类，并且分别重写了"发出声音"这个方法。

当我们使用多态性时，可以使用动物类的引用变量来引用狗类或猫类的对象。然后，通过这个引用变量调用"发出声音"的方法。由于狗类和猫类重写了该方法，所以在调用时实际执行的是各自子类中的方法。

这样，无论我们使用动物类的引用变量引用了狗类的对象还是猫类的对象，调用"发出声音"方法时都会根据对象的实际类型执行相应的方法，即狗类发出狗叫声，猫类发出猫叫声。这就是多态性的体现，通过统一的接口实现了不同类型对象的不同行为。

22. java中实现多态的机制是什么？

具体来说，多态性有以下两个关键要素：

  1. 继承关系（Inheritance）：多态性需要存在父类和子类之间的继承关系。子类可以继承父类的属性和方法，并且可以通过继承来扩展或修改继承的属性和方法。父类是一个通用的类型，而子类是具体的特殊类型。通过继承，子类可以继承和重用父类的代码。
  2. 方法重写（Method Overriding）：子类可以对父类中的方法进行重写。方法重写指的是在子类中重新定义和实现父类中已经存在的方法。通过方法重写，子类可以根据自身的特性来改变方法的行为，即使用子类的实现替代父类的实现。在多态性中，通过父类的引用变量来调用被重写的方法时，实际执行的是子类中重写的方法。

综合起来说，多态性利用继承和方法重写的机制，使得可以使用父类的引用变量来引用子类的对象，并且通过这个引用变量调用方法时，实际执行的是子类中的方法。这样可以在运行时根据对象的实际类型来确定方法的具体行为，实现了同一个方法在不同类型对象上的不同表现。多态性提高了代码的灵活性、可扩展性和可维护性，使得可以编写通用的代码来处理多种不同类型的对象。

23. abstract class和interface有什么区别?

含有abstract修饰符的class即为抽象类，abstract 类不能创建的实例对象。

含有abstract方法的类必须定义为abstract class，abstract class类中的方法不必是抽象的（也可以有具体方法）。abstract class类中定义抽象方法必须在具体(Concrete)子类中实现，所以，不能有抽象构造方法或抽象静态方法。如果子类没有实现抽象父类中的所有抽象方法，那么子类也必须定义为abstract类型。

接口（interface）可以说成是抽象类的一种特例，接口中的所有方法都必须是抽象的。接口中的方法定义默认为public abstract类型，接口中的成员变量类型默认为public static final。

下面比较一下两者的语法区别：

1）抽象类可以有构造方法，接口中不能有构造方法。

2）抽象类中可以有普通成员变量，接口中没有普通成员变量

3）抽象类中可以包含非抽象的普通方法，接口中的所有方法必须都是抽象的，不能有非抽象的普通方法。

4） 抽象类中的抽象方法的访问类型可以是public，protected和（默认类型,虽然

5）抽象类中可以包含静态方法，接口中不能包含静态方法

6）抽象类和接口中都可以包含静态成员变量，抽象类中的静态成员变量的访问类型可以任意，但接口中定义的变量只能是public static final类型，并且默认即为public static final类型。

24. abstract的method是否可同时是static,是否可同时是native，是否可同时是synchronized?

在Java中，抽象方法不能同时是静态（static）、本地方法（native）或同步方法（synchronized）。以下是对这三种情况的详细解释：

  1. 静态（static）抽象方法： 抽象方法是需要在子类中实现的方法，而静态方法是与类相关而不是与实例对象相关的方法。静态方法在类加载时就存在，并且可以通过类名直接调用，而抽象方法需要通过子类的实现才能调用。由于这两个特性的冲突，Java不允许定义静态抽象方法。
  2. 本地方法（native）抽象方法： 本地方法是指使用其他语言（如C或C++）实现的方法，需要通过Java的本地接口（JNI）与底层交互。抽象方法是需要在子类中实现的方法，而本地方法需要在其他语言中实现。因此，Java也不允许定义本地方法为抽象方法，因为无法在Java代码中提供本地方法的实现。
  3. 同步（synchronized）抽象方法： 抽象方法定义了方法的签名，而同步方法是在多线程环境下对方法进行互斥访问的机制。抽象方法的实现是在子类中进行的，而同步方法是通过对实例对象或类对象加锁来实现的。因为同步方法需要在具体的方法实现中指定加锁的对象，而抽象方法并没有具体的实现，所以也不允许将抽象方法声明为同步方法。

综上所述，抽象方法不能同时是静态的、本地方法的或同步方法的。抽象方法是一种规范，由子类提供具体的实现，而静态、本地方法和同步方法是针对具体的方法实现所定义的特性。

25.什么是内部类？Static Nested Class 和 Inner Class的不同。

内部类（Inner Class）是指在一个类的内部定义的另一个类。内部类与外部类之间存在一种特殊的关系，内部类可以访问外部类的成员，包括私有成员，而外部类也可以访问内部类的成员。内部类提供了一种封装和组织代码的方式，可以更好地实现软件设计的一些概念。

在Java中，内部类分为两种类型：静态内部类（Static Nested Class）和成员内部类（Member Inner Class）。

  1. 静态内部类（Static Nested Class）： 静态内部类是在外部类中用 static 关键字声明的内部类。静态内部类与外部类的关系类似于普通的静态成员与外部类的关系，它独立于外部类的对象，可以直接通过外部类名来访问。静态内部类不能访问外部类的非静态成员，只能访问外部类的静态成员。

以下是静态内部类的示例代码：

xxxxxxxxxx

    private static int outerData = 10;
    
    public static class StaticNestedClass {
        public void display() {
            System.out.println("Outer data: " + outerData);
        }
    }
    
    public static void main(String[] args) {
        OuterClass.StaticNestedClass nestedObject = new OuterClass.StaticNestedClass();
        nestedObject.display();
    }
}

在上述示例中，StaticNestedClass 是 OuterClass 的静态内部类。通过 OuterClass.StaticNestedClass 可以直接访问静态内部类，创建内部类的对象并调用其方法。

  1. 成员内部类（Member Inner Class）： 成员内部类是在外部类中直接定义的非静态内部类。成员内部类与外部类对象之间存在关联，它可以访问外部类的成员，包括私有成员。成员内部类的创建需要依赖于外部类的对象。

  • 创建静态内部类的对象不需要外部类的对象，而创建成员内部类的对象需要先创建外部类的对象。
  • 内部类提供了一种封装和组织代码的方式，能够更好地实现一些软件设计概念，如封装性、可重用性等。

String为什么不能被修改？

在Java中，String 类被设计为不可变类（Immutable Class），也就是说，一旦创建了 String 对象，它的值就不能被修改。

这是因为字符串在 Java 中广泛用于表示文本和字符串常量，并且在很多情况下，字符串在多个对象之间共享。为了确保字符串的不可变性，Java 采用了以下几个原因：

  1. 安全性：不可变性使得字符串对象在多线程环境下是安全的。由于字符串的值无法更改，不会发生竞态条件（Race Condition）和线程安全问题。多个线程可以同时访问和共享相同的字符串对象，而不会引发并发访问的问题。
  2. 缓存和性能优化：由于字符串的不可变性，Java 虚拟机可以在内部实现字符串常量池（String Pool）。字符串常量池是一个特殊的内存区域，用于存储字符串常量的唯一实例。当创建一个新的字符串对象时，如果字符串常量池中已经存在相同值的字符串，就直接返回常量池中的实例，而不是创建一个新的对象。这样可以节省内存空间，提高程序的性能和效率。
  3. 哈希值和安全散列：字符串的不可变性使得字符串的哈希值在创建时就可以被缓存起来，不需要每次重新计算。这在散列集合（如 HashMap）和散列算法中非常有用。如果字符串是可变的，其哈希值可能会发生变化，导致无法正确地进行查找和匹配。
  4. 可靠性和一致性：字符串的不可变性确保了其值在创建后不会被修改，保证了字符串对象的一致性和可靠性。这对于程序的正确性和预测性是至关重要的。

总的来说，String 类被设计为不可变的，这样可以提供安全性、性能优化、缓存、哈希值和一致性等优势。在日常开发中，推荐使用 String 类来表示字符串常量和不可变的字符串数据，而对于需要频繁修改的字符串操作，可以使用 StringBuilder 或 StringBuffer 类来提供可变性。

StringBuilder与String Buffer的区别？

StringBuilder 和 StringBuffer 是 Java 中用于处理可变字符串的两个类，它们具有相似的功能，但在线程安全性、性能和可变性方面存在一些区别。

主要区别如下：

  1. 线程安全性：
  • StringBuilder：StringBuilder 是非线程安全的。在多线程环境下，如果多个线程同时访问和修改同一个 StringBuilder 对象，可能会导致数据不一致或出现竞态条件。
  • StringBuffer：StringBuffer 是线程安全的。它的方法是同步的，通过使用同步锁来确保多个线程可以安全地访问和修改 StringBuffer 对象。因此，在多线程环境下，StringBuffer 是一个更安全的选择。
  1. 性能：
  • StringBuilder：StringBuilder 的方法没有被同步，因此在单线程环境下比 StringBuffer 更高效。它适用于单线程环境，当不需要考虑线程安全性，并且需要频繁进行字符串拼接或修改时，StringBuilder 的性能更好。
  • StringBuffer：StringBuffer 的方法是同步的，因此在多线程环境下比 StringBuilder 更安全，但也导致了一定的性能开销。如果在多线程环境下进行字符串操作，并且需要保证线程安全性，应该使用 StringBuffer。
  1. 可变性：
  • StringBuilder 和 StringBuffer 都提供了可变性，可以进行字符串的拼接、插入、删除、替换等操作。
  • StringBuilder 和 StringBuffer 都继承自 AbstractStringBuilder，但 AbstractStringBuilder 是包级私有的，不能直接使用。

综上所述，如果在单线程环境下进行字符串操作，建议使用 StringBuilder，它具有更好的性能。如果在多线程环境下进行字符串操作，或需要保证线程安全性，应该使用 StringBuffer。

至于为什么要把String类设计成不可变类，是它的用途决定的。其实不只String，很多Java标准类库中的类都是不可变的。在开发一个系统的时候，我们有时候也需要设计不可变类，来传递一组相关的值，这也是面向对象思想的体现。不可变类有一些优点，比如因为它的对象是只读的，所以多线程并发访问也不会有任何问题。当然也有一些缺点，比如每个不同的状态都要一个对象来代表，可能造成性能上的问题。所以Java标准类库还提供了一个可变版本，即StringBuffer。

字符串常量池（String Pool）是在 Java 堆内存中的一块特殊的内存区域，用于存储字符串常量的唯一实例。

在 Java 中，当创建字符串常量时，编译器会首先检查字符串常量池中是否存在相同值的字符串。如果存在，则直接返回常量池中的实例；如果不存在，则在常量池中创建一个新的字符串，并返回其引用。

字符串常量池的位置取决于 Java 虚拟机的具体实现。在传统的 JDK 实现中，字符串常量池位于 Java 堆内存的永久代（PermGen）或元空间（Metaspace）中，这取决于所使用的 JDK 版本。

在 JDK 7 及之前的版本中，字符串常量池位于永久代，永久代是一块用于存放类信息、常量池、静态变量等的内存区域。但是，在 JDK 8 及之后的版本中，永久代被移除，取而代之的是元空间，字符串常量池也被迁移到了元空间中。

无论字符串常量池的具体位置是永久代还是元空间，它都是在 Java 堆内存中的一部分。这是因为 Java 堆内存用于存储所有的对象实例和数组，并且字符串常量池中的字符串实例也是对象。因此，它们都受到 Java 堆内存的管理。

需要注意的是，从 JDK 9 开始，字符串常量池的实现发生了变化，字符串常量池被移出到堆中的一部分，这样就不再占用永久代或元空间的内存空间。

总结：字符串常量池是 Java 堆内存中的一块特殊区域，用于存储字符串常量的唯一实例。它的具体位置在 JDK 的不同版本中有所不同，传统的 JDK 实现中位于永久代或元空间，从 JDK 9 开始则位于堆中的一部分。

40.运行时异常与一般异常有何异同？

运行时异常（RuntimeException）和一般异常（Checked Exception）在 Java 中有以下几个区别：

  1. 异常检查：
  • 运行时异常：运行时异常是不受编译器强制检查的异常，即编译器不会强制要求在代码中处理或声明运行时异常。程序可以选择捕获并处理运行时异常，也可以选择将其继续抛出。常见的运行时异常包括 NullPointerException、ArrayIndexOutOfBoundsException、ClassCastException 等。
  • 一般异常：一般异常是受编译器强制检查的异常，编译器会强制要求在代码中进行异常处理或在方法签名中声明可能抛出的异常。如果没有处理或声明一般异常，编译器将报错。常见的一般异常包括 IOException、SQLException、ClassNotFoundException 等。
  1. 异常处理：
  • 运行时异常：对于运行时异常，程序可以选择捕获并处理，也可以不处理。如果未处理，运行时异常将导致程序的异常终止，堆栈跟踪信息将被输出。
  • 一般异常：一般异常必须显式处理，要么通过捕获并处理异常，要么通过在方法中使用 throws 关键字声明可能抛出的异常。如果不进行处理，编译器将报错，并指示缺少异常处理代码。
  1. 编程建议：
  • 运行时异常：通常情况下，运行时异常是由程序错误导致的，如空指针引用、数组越界等。对于这类异常，应该通过编码规范和良好的程序设计来避免它们的发生。捕获和处理运行时异常主要用于程序的健壮性和调试目的。
  • 一般异常：一般异常通常是由外部因素或业务逻辑引起的，如文件读写错误、数据库连接问题等。对于这类异常，应该进行适当的异常处理，以确保程序在遇到异常情况时能够进行合理的操作，如关闭资源、回滚事务等。

需要注意的是，运行时异常和一般异常都是从 Java 内建的 Throwable 类继承而来，它们都是异常的子类，只是在异常检查和处理上有所不同。

简单总结是：运行时异常是不可提前知的，一般异常是可提前预知的。

总结：运行时异常和一般异常在异常检查和处理上有区别。运行时异常不受编译器强制检查，可以选择捕获和处理，也可以选择继续抛出；一般异常受编译器强制检查，必须进行显式处理。编程中应避免发生运行时异常，而对于一般异常应进行适当的处理。

41.error和exception有什么区别?

在 Java 中，Error 和 Exception 都是 Throwable 类的子类，表示在程序执行过程中可能出现的问题。它们之间的主要区别在于它们的使用场景和处理方式：

  1. Error：
  • Error 表示严重的问题，通常是无法恢复的错误，它们往往是由 JVM 或底层系统引起的，例如 OutOfMemoryError、StackOverflowError 等。Error 通常是不可捕获和处理的，因为它们表示 JVM 或底层系统发生了无法恢复的问题，程序无法继续正常执行。当发生 Error 时，通常会导致程序异常终止，并输出错误信息。
  1. Exception：

  • Exception 表示非正常情况，但通常是可预料和可处理的问题，它们可以由程序员编写的代码引起，也可以由外部因素或底层库引起。Exception 又分为两种类型：

    • Checked Exception（一般异常）：Checked Exception 是需要在代码中进行显式处理的异常。它们继承自 Exception 类，但不继承自 RuntimeException。Checked Exception 包括 IOException、SQLException 等，它们通常表示外部资源访问或操作失败，程序需要处理这些异常情况，如捕获并处理异常，或通过在方法中使用 throws 关键字声明可能抛出的异常。
    • Unchecked Exception（运行时异常）：Unchecked Exception 是不需要在代码中进行强制处理的异常。它们继承自 RuntimeException 类，通常由程序错误或错误的使用方式引起。运行时异常包括 NullPointerException、ArrayIndexOutOfBoundsException、ClassCastException 等。程序可以选择捕获和处理运行时异常，也可以选择不处理，未处理的运行时异常将导致程序异常终止，并输出堆栈跟踪信息。

总结：

  • Error 表示严重的问题，通常是无法恢复的错误，由 JVM 或底层系统引起，无法捕获和处理。
  • Exception 表示可预料和可处理的非正常情况，分为 Checked Exception 和 Unchecked Exception。Checked Exception 需要在代码中进行显式处理，而 Unchecked Exception 可选择是否处理。

stop() 和 suspend() 方法不推荐使用的原因如下：

  1. stop() 方法：

    • stop() 方法用于立即停止线程的执行。它会导致线程立即终止，不会执行任何清理工作，也不会释放占用的资源。这可能导致程序处于不一致的状态，无法预料的结果。
    • stop() 方法容易引发死锁情况。当一个线程在执行关键操作时被突然终止，可能会使得对象的状态处于不一致的状态，导致其他线程在访问该对象时出现问题。
    • 由于线程突然终止，可能会导致未完成的操作无法正确释放资源，例如未关闭的文件、网络连接等，进而导致资源泄漏。

  2. suspend() 方法：

    • suspend() 方法会暂停线程的执行，将线程挂起，并且不会释放占用的资源。这可能导致其他线程在访问被挂起的线程占用的资源时出现阻塞或死锁。
    • 当一个线程被挂起时，如果其他线程需要访问被挂起线程的共享资源，可能会出现资源竞争和不确定的结果。
    • suspend() 方法容易引发死锁情况。如果一个线程在执行关键操作时被挂起，可能会使得其他线程在等待被挂起线程释放资源时陷入死锁状态。

    由于上述问题，stop() 和 suspend() 方法已被标记为过时（deprecated），不推荐使用。推荐使用更安全、可控的方式来管理线程的执行，例如使用共享标志或状态来控制线程的停止、暂停和恢复操作，或者使用interrupt() 方法来请求线程停止或中断。这样可以更好地避免潜在的线程安全问题和资源泄漏。

45.sleep() 和 wait() 有什么区别?

sleep() 和 wait() 是 Java 中用于线程控制的方法，它们有以下区别：

  1. 来自不同的类：

    • sleep() 是 Thread 类的静态方法，通过 Thread.sleep() 调用。
    • wait() 是 Object 类的方法，通过对象实例调用。

  2. 使用的上下文：

    • sleep() 通常用于线程休眠，即暂时停止当前线程的执行，不释放对象锁，让其他线程有机会执行。它在定时任务、模拟延迟等场景中常被使用。
    • wait() 用于线程间的协作与通信，即线程等待某个条件的满足，同时释放对象的锁，以便其他线程可以获得锁并执行。它在多线程协作、线程安全的同步操作中常被使用。

  3. 调用的方式：

    • sleep() 可以直接通过 Thread.sleep() 调用，指定线程休眠的时间段，以毫秒为单位，也可以使用带有纳秒参数的重载方法。
    • wait() 需要在对象的同步代码块（synchronized block）或同步方法（synchronized method）中调用，使用 object.wait() 进行调用。调用 wait() 后，线程会进入等待状态，直到其他线程调用对象的 notify() 或 notifyAll() 方法来唤醒等待线程。

  4. 锁的释放：

    • sleep() 方法在执行期间不会释放线程持有的锁。
    • wait() 方法会释放线程持有的对象锁，使得其他线程可以获得该对象的锁并执行。

  5. 唤醒方式：

    • sleep() 方法在指定的时间到达后，线程会自动苏醒，继续执行。
    • wait() 方法需要等待其他线程调用对象的 notify() 或 notifyAll() 方法来唤醒等待的线程。

总结：

  • sleep() 用于线程休眠，不释放对象锁，常用于定时任务和模拟延迟。
  • wait() 用于线程间的协作与通信，释放对象锁，常用于多线程协作和线程安全的同步操作。

    它们是 Object 类的方法，用于线程的协作与通信。wait() 方法使当前线程进入等待状态，同时释放对象的锁，而 notify() 方法用于唤醒等待中的线程。

wait() 和 notify() 的典型用法是在共享对象上进行同步操作，通过等待和唤醒的方式实现线程之间的协作。以下是一个示例：

xxxxxxxxxx
public class SyncExample {
    private Object lock = new Object();
    private boolean isReady = false;


    public void producer() {
        synchronized (lock) {
            while (isReady) {
                try {
                    lock.wait(); // 等待消费者线程通知
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
            // 执行生产操作
            System.out.println("Producer: Producing data...");
            isReady = true;
            lock.notify();
        }
    }


    public void consumer() {
        synchronized (lock) {
            while (!isReady) {
                try {
                    lock.wait(); // 等待生产者线程通知
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
            // 执行消费操作
            System.out.println("Consumer: Consuming data...");
            isReady = false;
            lock.notify();
        }
    }


    public static void main(String[] args) {
        SyncExample syncExample = new SyncExample();


        Thread producerThread = new Thread(() -> { // 利用lambda表达式创建生产者线程
            syncExample.producer();
        });


        Thread consumerThread = new Thread(() -> { // 利用lambda表达式创建消费者线程
            syncExample.consumer();
        });


        producerThread.start();
        consumerThread.start();
    }
}

在上面的示例中，SyncExample 类包含了一个共享对象 lock 和一个布尔标志 isReady。生产者线程通过获取锁、检查 isReady 的状态并等待，直到消费者线程通知可以生产。消费者线程通过获取锁、检查 isReady 的状态并等待，直到生产者线程通知可以消费。通过这种方式，生产者和消费者线程之间实现了同步。

需要注意的是，wait() 和 notify() 必须在同步代码块内部使用，并且对应同一个对象的监视器（即锁）进行调用。否则，会导致 IllegalMonitorStateException 异常。

综上所述，wait() 和 notify() 是实现线程之间同步和协作的重要工具，可用于共享对象上的等待和唤醒操作。

无论是使用匿名内部类还是 lambda 表达式，它们都实现了 MyInterface 接口的抽象方法，并且在调用时输出了相应的语句。这展示了匿名内部类和 lambda 表达式的用法和简洁性。

48.启动一个线程是用run()还是start()？

启动一个线程是调用start()方法，使线程就绪状态，以后可以被调度为运行状态，一个线程必须关联一些具体的执行代码，run()方法是该线程所关联的执行代码。
